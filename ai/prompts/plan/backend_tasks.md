# 后端任务列表 (B1-B11)

## 技术栈要求
- Node.js 18+
- Express.js 框架
- TypeScript 开发
- JWT 用于认证
- Multer 用于文件上传
- Prisma 或 Drizzle ORM
- PostgreSQL/SQLite 数据库

## 任务列表

### B1: 初始化后端项目
**输入**: 无
**输出**: `app/backend/` 目录包含基础项目结构（package.json、tsconfig.json、基础Express应用）
**依赖关系**: 无
**说明**: 创建Express应用骨架，配置TypeScript和基础中间件

### B2: 实现用户认证逻辑（注册/登录）
**输入**: B1 的输出，数据库D1设计完成
**输出**: `/api/auth/register` 和 `/api/auth/login` API端点
**依赖关系**: B1, D1
**说明**: 实现用户注册（密码加密）和登录（JWT签发）逻辑

### B3: 实现认证中间件
**输入**: B2 的输出
**输出**: 验证JWT的Express中间件，保护路由功能
**依赖关系**: B2
**说明**: 创建authMiddleware，验证请求头中的Bearer Token

### B4: 实现照片上传API
**输入**: B1 的输出，数据库连接D2完成
**输出**: `/api/photos/upload` API端点，支持multipart/form-data
**依赖关系**: B1, D2
**说明**: 使用Multer处理文件上传，保存文件到本地或云存储，保存元数据到数据库

### B5: 实现照片删除API
**输入**: B4 的输出
**输出**: `/api/photos/:id` DELETE端点
**依赖关系**: B4
**说明**: 根据ID删除照片文件和数据库记录，需要认证

### B6: 实现照片列表API
**输入**: 数据库连接D2完成
**输出**: `/api/photos` GET端点，支持分页
**依赖关系**: D2
**说明**: 获取用户上传的照片列表（需认证），返回分页数据

### B7: 实现公开照片列表API（用于展示页）
**输入**: 数据库连接D2完成
**输出**: `/api/photos/public` GET端点，无需认证
**依赖关系**: D2
**说明**: 获取所有公开照片，用于三个展示页面

### B8: 配置静态文件服务
**输入**: B4 的输出
**输出**: Express静态文件服务，可通过URL访问上传的图片
**依赖关系**: B4
**说明**: 配置Express的静态文件中间件，服务uploads目录

### B9: 错误处理与日志
**输入**: 所有API端点
**输出**: 统一的错误响应格式，请求日志中间件
**依赖关系**: 所有后端任务
**说明**: 创建错误处理中间件，统一错误格式，添加请求日志

### B10: API文档（OpenAPI/Swagger）
**输入**: 所有API端点
**输出**: 可访问的API文档端点（如 `/api-docs`）
**依赖关系**: 所有后端任务
**说明**: 使用Swagger UI或类似工具生成API文档

### B11: 后端测试（单元与集成）
**输入**: 所有API端点
**输出**: 测试用例，CI集成配置
**依赖关系**: 所有后端任务
**说明**: 使用Jest/Supertest编写API测试，配置测试环境

## 执行建议
1. 先完成B1基础结构，再实现具体功能
2. 认证逻辑（B2,B3）应在文件上传（B4,B5）之前完成
3. 静态文件服务（B8）需要上传功能（B4）完成后配置
4. 错误处理和文档（B9,B10）可在核心API完成后添加
5. 测试（B11）应在所有功能完成后进行
