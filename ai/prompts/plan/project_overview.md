# 项目概述 - 个人照片分享网站

## 项目目标
开发一个交互式个人照片分享网站，包含四个展示页面和一个管理页面。

## 核心功能
1. **环形展示页面** - 图片圆形轨道展示，支持滚动旋转交互
2. **3D轮播页面** - 3D透视轮播画廊，自动循环与景深效果  
3. **泡泡头像筛选页面** - 自由浮动头像，悬停筛选与视差滚动
4. **照片管理页面** - 支持照片上传、删除等操作（需登录）

## 技术要求
- **前端**: React + TypeScript + Vite，动画使用 Framer Motion/Three.js
- **后端**: Node.js + Express，JWT认证，Multer文件上传
- **数据库**: PostgreSQL (开发可用SQLite)，Prisma/Drizzle ORM
- **部署**: Vercel + Supabase 或 Railway/Render

## 开发原则
1. 每个任务独立可执行，有明确输入输出
2. 任务拆解到可直接编码的粒度
3. 前端先实现静态页面，再添加动画和交互
4. 后端先实现基础API，再集成认证和文件处理
5. 数据库设计先行，为前后端提供数据模型

## 模块划分
- **前端模块**: 23个任务 (F1-F23)
- **后端模块**: 11个任务 (B1-B11)  
- **数据库模块**: 4个任务 (D1-D4)
- **部署模块**: 7个任务 (O1-O7)

## 执行阶段
1. **阶段1 - 基础搭建** (1-2天): 初始化项目，配置环境，创建基础结构
2. **阶段2 - 核心展示页面** (3-5天): 实现三个展示页面的UI与交互
3. **阶段3 - 后端API与集成** (2-3天): 实现认证、文件上传、API集成
4. **阶段4 - 照片管理页面与完善** (1-2天): 完成管理页面，样式优化
5. **阶段5 - 测试与部署** (1-2天): 测试、构建、部署

## 文件结构说明
```
project/app/frontend/     # 前端代码
project/app/backend/      # 后端代码  
project/app/database/     # 数据库设计和迁移
project/ai/prompts/       # 本目录，包含所有任务提示词
```

## 使用说明
每个任务提示词包含：
- **任务ID**: 唯一标识符 (如 F1, B2)
- **任务名称**: 简洁描述
- **输入**: 完成任务所需的输入（文件、配置、依赖任务输出）
- **输出**: 完成后的可交付成果（文件、功能、API端点）
- **依赖关系**: 需要先完成的任务ID

按照 `execution_sequence.md` 中的顺序执行任务。
